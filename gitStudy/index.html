git命令学习

git 命令学习

本地库操作

分支/提交

  1. 生成本地分支

  • git checkout -b 分支名

  • git switch -c 分支名

  1. 切换分支

  • git switch 分支名

  • git checkout 分支名

  1. 查看分支

  • git branch

  • git branch -vv 查看分支和分支当前状态

  1. 查看当前文件状态

  • git status 查看当前文件修改状态

  • git diff 查看具体修改内容

  1. 暂存修改
  • git add 文件名，git add . 为暂存所有修改（一般直接使用git add .）
  1. 提交

  • git commit -m “提交描述”

  1. 合并分支

  • git merge 上述commit操作是在b3分支执行，现在在b1分支同步b3修改，是的b1也包含b3修改；先切换b1分支，再执行git merge

  1. 变基

  • git rebase 基点分支名，rebase操作会先提取共同改的parents,然后将待变基分支最新提交复制嫁接到基点分支最新提交之后

  1. 撤销提交

  • git revert [提交hash值] 直接撤销一个提交，并且会新增一条提交来记录撤销操作，便于团队协作。

  • git revert –no-edit [提交hash值] 不会打开vim编辑器要求编辑的命令(直接进行撤销和提交)

  • git revert –no-commit [提交hash值] 此时会撤销提交，但是不会进行commit操作，此时撤销的修改会是暂存的状态，可以撤销多条提交，最后手动进行提交。

修改文件管理（放弃,存储）

  1. 放弃当前管理中的修改
  • 未被add的修改可以通过

    • git checkout [文件路径]

    • git restore [文件路径]

  • 已被add的修改

    • git restore –staged [文件路径] 注意此时这条操作只会把暂存中的文件取出来，将文件状态变更为未暂存的状态。此时修改依然存在，如果需要彻底撤销还需执行一次git restore [文件路径]

  1. 存储变更

  • git stash save ‘[存储注释]’ 用于不想暂存文件,但是想保留进行的修改。一般在切分支保留上一个分支修改，以便回头继续进行修改。

  • git stash list 查看存储列表

  • git stash show 查看第一个(最近的)存储起来的更改。如果想要查看指定stash变更,就在后面stash@{[num]},num为stash list显示的序号(注意如果不在gitBash窗口下小括号要转义)。如果后面再跟 -p 就可以显示具体的修改。

  • git stash pop 弹出第一个(最近的)存储并删除,如果要弹出指定存储在后面更 stash@{[num]} num为stash list显示的序号

  • git stash apply 应用第一个(最近的)存储但是不删除stash list 中的记录

  • git stash drop stash@{[num]} 删除指定存储

  • git stash clear 一次性清除存储列表中所有存储记录。
